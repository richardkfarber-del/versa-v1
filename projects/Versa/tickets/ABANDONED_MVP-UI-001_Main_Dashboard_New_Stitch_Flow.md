### 🎫 TICKET: MVP-UI-001
**Title:** Main Dashboard & "New Stitch" Initialization Flow
**Assignee:** Green Lantern
**Priority:** Highest / P0
**App:** Google Stitch (MVP)

**Objective:**
Design the primary landing experience (Main Dashboard) for returning users and the initial creation flow for starting a new "Stitch" project. This is the foundational UI that users will interact with immediately after authentication.

**Design Requirements & Scope (MVP):**
1. **Main Dashboard Screen:**
   - **Google Material Design 3 Alignment:** Ensure the UI aligns with Google's standard design language (clean, spacious, rounded corners, standard typography).
   - **Recent Stitches Grid/List:** A section displaying previously created projects with thumbnail placeholders, project titles, and "last edited" timestamps.
   - **Empty State:** A friendly, intuitive empty state for first-time users guiding them to create their first Stitch.
   - **Primary CTA:** A prominent Floating Action Button (FAB) or central CTA for "Create New Stitch".

2. **"New Stitch" Initialization Flow (Modal or New Screen):**
   - **Project Setup:** Input fields for "Stitch Name" and "Description" (optional).
   - **Initial Source Selection:** A streamlined UI component to select the base media/documents (e.g., "Upload from Device" or "Select from Google Drive" placeholders).
   - **Action Buttons:** "Cancel" and "Create Stitch".

**Deliverables Required (For Local Handoff):**
- High-fidelity wireframes/mockups for the screens mentioned above.
- Exported as standard image formats (PNG or JPG) or a consolidated PDF.
- (Optional but preferred) A brief markdown document (`MVP-UI-001-notes.md`) detailing color hex codes, typography choices, and any specific component states (hover, active, disabled).

**Handoff Instructions:**
Save all exported assets and notes into a local directory named `Stitch_MVP_UI_001` and notify the team when the files are ready for review and integration.
